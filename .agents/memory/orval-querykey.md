---
name: Orval v8 queryKey requirement
description: All useGetX hooks generated by Orval v8 require explicit queryKey in query options.
---

All `useGetX` hooks generated by Orval require an explicit `queryKey` field in their `query` option object, otherwise TypeScript throws `TS2741: Property 'queryKey' is missing`. 

**Rule:** Always import and pass the corresponding `getGetXQueryKey()` function:

```ts
import { useGetFeed, getGetFeedQueryKey } from "@workspace/api-client-react";

const { data } = useGetFeed(undefined, {
  query: { queryKey: getGetFeedQueryKey(), enabled: isAuthenticated },
});
```

**Why:** Orval v8 changed the generated hook signatures to require queryKey for type safety and React Query v5 compatibility.

**How to apply:** Any time you add a new `useGetX` hook call, always import and pass `getGetXQueryKey(...)` (matching positional args if the query takes params).
